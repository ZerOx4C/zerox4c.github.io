HexoとGitHub Pagesでブログを始めよう

ブログ放置マンでも続けられるような環境をGitHub PagesとHexoで構築しました。

環境選び

今まで何度かブログを放置してきて思い当たる原因を挙げてみると、こんな感じでした。

原因 対策
役割がTwitterとカブる 内容を技術メモに絞る
サーバのお世話に終始してしまう GitHub Pagesを使う
書くのが面倒 Markdownが使えるようにする

内容を技術メモに絞るのはそのように運用しましょう、頑張ります。

あとはGitHub Pagesで動いてMarkdownが使えるシステム探しです。
こちらを参考にしました。

Github Pagesでブログ構築ができる静的サイトジェネレーター総まとめ - Qiita

Hexoを導入して動かすまで

テーマとデプロイについてのみ補足しますが、それ以外は先人のありがてぇ記事をご覧ください。
すっごい簡単でした。

Hexo と GitHub Pages でブログ環境を構築して公開する - Qiita

ちなみにhexo initした際に.gitignoreも作られてるので、そのままgit initしてリポジトリ管理できます。

私の場合はデプロイ先はGitHub Pages用のリポジトリで、hexo initしたディレクトリをBitbucketのプライベートリポジトリに置いてます。

テーマをカスタムしたい人向け補足

テーマはcloneすれば動いてくれますが、その場合submoduleになるため、オレオレカスタムを施した際の差分をどうするか困りました。

forkしたリポジトリをcloneするか、cloneしたフォルダのgit管理をやめて直下に置くか、あたりの方法が取れそうですが、前者は面倒、後者は元テーマの変更が一切追えないので微妙です。

そんな中、まさに探していた！という記事がこちらに。

submoduleとsubtree-mergingの使い分け - Qiita

  • 外部リポジトリを外のものとして取り込むならサブモジュール
  • 外部リポジトリを取り込みつつ、中のものとして手を加えていくならサブツリーマージ
  • git初心者にはサブモジュール(異論ありそう)

ということで、今回はサブツリーマージを使ってみることにしました。
とりあえずテーマを導入するときはこんな感じ。

詳しい使い方はこちらを参照してください。

Git - サブツリーマージ

デプロイできぬ人向け補足

先人の指示通りやってdeployで何度かコケたのでメモ。

Deployer not found

version 3からはdeployerをインストールする必要があるみたいです。

ERROR Deployer not found: github · Issue #1040 · hexojs/hexo

Permission denied (publickey).

_config.ymlのrepoにsshで記述すると、SSH Keyが未登録だと弾かれます。

HTTPSに直すか、公開鍵を登録しておきましょう。

まとめ

  • GitHub PagesとHexoで簡単にMarkdownブログが始められるぞ
  • テーマをforkするのが面倒ならサブツリーマージすると良いかも
